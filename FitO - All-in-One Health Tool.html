FitO - All-in-One Health Tool

About

FitO is an all-in-one health Android app which helps monitor our daily lifestyle. Several features of it include:

  • Tracking your footsteps
  • Monitor your daily diet - one can add food items they have eaten in their daily routine and monitor their calorie intake and other nutrients
  • Can track calories - eaten and burnt with daily exercises
  • Keep a check on your water intake
  • Monitor your sleep cycle
  • and keeps you motivated

It also provides the user with their personal Health Diary, where the user can take a peek into their fitness history and accounts for all the activities for a particular day (just like a daily diary entry!). Also, it provides Statistical analysis of the activities on a weekly and monthly basis with the help of charts where one can monitor their overall activity from the past and try to improve.

App Architecture

The Architecture used in this app is Model-View-Controller (MVC). Let’s take a glance at the file structure:

  • Model - We can see the Helper database classes, which populate the data like get data, set data etc., are separated from others.
  • View - All the layouts are separated from other classes and help display to the user on the screen.
  • Controller - The brains of the system, each segment of the app is constituted in a particular folder to organize the classes in a better way.

Other Android Features:

  • Components built using Fragments: To improve reusability, almost all of the views are made using Fragments which also helps in the easy navigation of tabs.
  • Use of Nav Controller: We have used the Nav controller to navigate through the bottom tabs or any other fragments connected.
  • Navigation view

  • Database used - Firebase
    • We are storing all users' data by creating a unique node for each user. The node will further contain nodes such as steps, activity, food_intake, etc., which have data stored in nodes of date. This is the overall structure of the database.

  • Several other features of the app:
    • Login via Google OAuth 2.0
    • Push Notifications - There will be notifications thrice at specific times of the day. The notifications remind the user to drink water and mention the same in the app.
    • Motion Sensors - to calculate the number of steps walked
    • APIs include - EDAMAM API for Food databases and Recipe so that users can easily add to the food consumed tab.
